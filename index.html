项目背景

虚拟制片场景中的实时控制挑战

😫
传统方式痛点
美术人员需在 UE5 编辑器中手动调整灯光、全息、特效等参数，效率低下且无法与 AI 协作，难以应对实时演出场景的快速变化需求。
💡
核心洞察
C++ 只管"传值"，蓝图管"执行"。就像 Widget 滑条的 On Value Changed 事件一样，参数变化应触发蓝图事件，由蓝图决定如何响应。
🎯
设计目标
构建一个 AI 可理解、人类可操控、UE5 可执行的播控中台，实现"AI 组织逻辑，人类触发执行，UE5 渲染呈现"的分层协作。
⚡
关键约束
UE5 是参数唯一源头，AI 禁止直接控场，只做功能封装。单路径数据流：前端/AI → Python UDP → UE5，无旁路。

MCP AI 工具

AI 与播控中台的桥梁

工具名 参数 用途
get_capabilities 无 获取 UE5 全量参数清单 + 已有 Feature，AI 必须先调用此工具
compose_feature name, description, param_names AI 将 UE5 参数封装为功能 Feature，自动推送到前端看板
configure_layout feature_order, collapsed, highlights AI 编排前端画布布局，控制卡片排序、折叠、高亮
set_float_parameter param_name, value 精确参数控制，AI 快捷调试通道（禁止高频调用）
execute_stage_cue cue_name, value, target 智能导播指令，语义解析 → 控前端/控 UE5
⚠️ 重要约束
AI 使用这些工具必须完全由用户授权，不能自动执行！AI 绝对不能自动执行任何控场操作，只能根据用户需求组织播控逻辑。

核心设计原则

约束即自由，边界即架构

01. UE5 是参数唯一源头
参数名、类型、默认值、范围、标签全部由 UE5 VPAgent| 属性定义。Python 端不定义参数，只做转发和封装。
02. AI 禁止直接控场
AI 绝对不能自动执行任何控场操作，只能根据用户需求组织播控逻辑。任何实际播控操作必须由用户或前端触发。
03. AI 只做功能封装
AI 读取 UE5 参数 → 按逻辑组合为 Feature → 参数值用 UE5 默认值。AI 不创造参数值，只命名和组织。
04. 事件驱动执行
C++ 不自动执行业务逻辑，蓝图在事件中决定做什么。C++ 只管"传值"，蓝图管"执行"。
05. 单路径数据流
前端/AI → Python UDP → UE5，无旁路。无双向同步，无参数持久化，所有状态在内存中。
06. 8字节 UDP 协议
uint32 FNV-1a Hash + float32 Value，大端序。极致轻量，确保实时性。

蓝图使用指南

三种方式，由轻到重

1
VPAgentComponent + Bind Event
最轻量 · 推荐

1. 创建自定义 Actor
2. 添加 VP Stage Agent 组件
3. Bind Event to On Control Command
4. 在事件中写自定义逻辑

✓ 纯逻辑组件，无场景组件
2
继承 VPStageEffectActor
带预设参数 · 示例

1. 创建蓝图子类继承 VPStageEffectActor
2. Override On Parameter Changed
3. 使用 ApplyAllParameters() 默认行为
4. 或写自定义逻辑

✓ 自带全息/灯光组件
3
添加自定义可控参数
零代码扩展

在蓝图子类中添加新变量，设置 Category 为 VPAgent|你的分类，Subsystem 自动发现并纳入 UDP 控制：

UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "VPAgent|灯光控制", meta = (ClampMin = "0.0", ClampMax = "1.0", VPA_Tags = "亮度,发光,Emissive", VPA_Description = "控制舞台主灯亮度")) float My_New_Param = 0.5f;
