## 1. 竞品概览

| 产品 | 产品类型 | 核心定位 |
|------|---------|---------|
| **Pixotope** | 完整虚拟制片平台 | 基于 Unreal 引擎的一站式虚拟/AR/XR 生产方案 |
| **Ross Video Carbonite** | 广电专业切换台 | 传统电视/现场直播视频信号切换与控制 |
| **Notch** | 实时内容创作平台 | 节点式实时渲染/AR/VR/媒体服务器控制 |
| **VPBroadcast** | 轻量播控插件 | 基于 Unreal 插件的 UDP/MCP 播控方案（我们的方案） |

## 2. 详细对比

### 2.1 架构与核心设计理念

| 维度 | Pixotope | Ross Video Carbonite | Notch | VPBroadcast |
|------|---------|---------------------|-------|------------|
| **渲染引擎** | 定制 Unreal Engine 5.5.3 | 自有硬件/固件渲染 | 自有 NURA 渲染架构（路径追踪+光栅化） | 原生 Unreal 5.1 插件 |
| **参数定义位置** | Unreal 项目 + 蓝图 | 切换台硬件/固件 | Notch Builder 项目/节点 | Unreal 项目 + 蓝图（`VPAgent|` 属性） |
| **执行层** | 蓝图/Unreal Timeline/Sequencer | 切换台硬件/固件 | Notch 节点图/时间线 | 蓝图（事件驱动） |
| **核心设计** | WYSIWYG 预览 + Director UI 控制 | 硬件按钮/推子控制 + 视觉反馈 | 节点式创作 + Web Remote 控制 | C++ 轻量框架 + 蓝图自由执行 |
| **对齐度** | 🟢 **完美对齐**我们的设计理念 | 🟡 **部分对齐** | 🟢 **完美对齐** | 🟢 **符合理念** |

### 2.3 通信协议与可靠性

| 维度 | Pixotope | Ross Video Carbonite | Notch | VPBroadcast |
|------|---------|---------------------|-------|------------|
| **连续参数协议** | 自定义 API + DMX/Art-Net | 硬件专用协议 | OSC/Art-Net/MIDI | UDP 8B (单向，最新值胜出) |
| **事件触发协议** | MOS/API 带确认 | 硬件脉冲（AUTO TRANS 按钮） | OSC/API 带确认 | 我们当前 UDP（需优化为自动复位） |
| **状态同步机制** | 可选 API 拉取，但主要看 PGM 监视器 | 面板显示当前状态，主要看 PGM | Web Remote 实时更新，主要看 PGM | 我们当前无，可加快照回传 |
| **视觉反馈为主** | 🟢 是，PGM 预览 + 监视器 | 🟢 是，PGM/PVW 监视器 | 🟢 是，主要看输出 | 🟢 是，主要看 UE 监视器 |

## 3. 我们 VPBroadcast 的优势与优化建议

### 3.1 优势

| 优势 | 说明 |
|------|------|
| **极低成本** | 纯软件 Unreal 插件，无需专业硬件 |
| **架构简单轻量** | 仅核心通信与事件触发，无冗余 |
| **完美对齐 UE 生态** | 直接在 UE 蓝图中实现执行逻辑，学习成本低 |
| **参数在 UE 定义** | 符合 Pixotope/Notch/Carbonite 的行业最佳实践 |
| **视觉闭环为主** | 符合广电行业“看监视器调”的习惯 |

### 3.2 优化建议（优先级排序）

| 优先级 | 建议 | 实现成本 | 说明 |
|--------|------|---------|------|
| **P0（关键）** | 事件发送自动复位 | 极低 | 解决事件卡死风险，对齐行业标准（见第 2.5 节） |
| **P1（重要）** | 加 UDP 快照回传 | 低 | 可选功能，修正前端显示，视觉仍是主要反馈 |
| **P2（可选）** | 加 TCP 备用通道 | 中 | 仅在跨网络高可靠场景需要，本地 UDP 已足够 |

## 4. 结论：我们的方案符合行业最佳实践

通过对 Pixotope、Ross Video Carbonite 和 Notch 的分析，我们可以得出：

| 结论 | 依据 |
|------|------|
| **参数在渲染端定义是对的** | Pixotope/Carbonite/Notch 都是这么做的 |
| **C++ 只传值，蓝图管执行是对的** | 完美对齐所有竞品的设计 |
| **过渡曲线在渲染端预设是对的** | 符合广电/虚拟制片的行业标准 |
| **连续参数用 UDP 是对的** | 和 OSC/DMX/Art-Net 一致，最新值胜出 |
| **视觉反馈为主是对的** | 和广电/虚拟制片的工作流一致 |

我们只需要两个小优化即可达到行业标准：
1. 事件发送自动复位（解决卡死风险）
2. 可选加 UDP 快照回传（修正前端显示）
